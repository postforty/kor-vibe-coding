# 마크다운 기반 Notion 스타일 WYSIWYG 에디터 구현 계획 (수정됨)

사용자님의 피드백을 반영하여 화면 분할 방식이 아닌, **Notion처럼 글을 쓰는 즉시 렌더링되는 단일 화면(WYSIWYG) 방식**으로 구현 계획을 전면 수정하였습니다.

## 핵심 요구사항 및 변경점
1. **작성 즉시 렌더링 (WYSIWYG)**: 
   - 화면을 반으로 나누지 않습니다. `# `를 치고 스페이스바를 누르면 즉시 제목(H1)으로 변환되는 등 Notion과 완전히 동일한 UX를 제공합니다.
   - 단일 화면이므로 양쪽 창의 "스크롤 동기화" 문제는 자연스럽게 해결됩니다. (내가 글을 쓰고 있는 위치에 스크롤이 머물게 됨)
2. **로컬 디스크 저장 (답변)**: 
   - 네, 작성된 문서는 브라우저나 가상 공간이 아닌 **사용자님의 실제 PC(로컬 디스크)에 `.md` 파일 형태로 저장**됩니다. 프로젝트 폴더 내에 `documents` (또는 `data`) 폴더가 자동 생성되어 관리됩니다.
3. **현재 폴더(kor-vibe-coding)에 즉시 초기화**:
   - 하위 폴더를 만들지 않고 현재 작업 중인 공간에 바로 Next.js 환경을 구축합니다.

---

## 🛠 수정된 기술 스택 (Technology Stack)
- **프레임워크**: `Next.js` (App Router)
  - 웹 UI 구동 및 로컬 PC의 파일 시스템(`fs`) 접근을 위한 API 역할을 수행합니다.
- **WYSIWYG 에디터**: `@blocknote/react` (또는 `Tiptap`)
  - Notion 스타일의 블록 기반 에디터를 구현하는 최적의 라이브러리입니다.
  - 슬래시(`/`) 커맨드, 블록 드래그 앤 드롭, 마크다운 단축키를 기본 지원하며 마크다운 텍스트로의 변환(Import/Export)이 자유롭습니다.
- **스타일링**: `Tailwind CSS` + 순수 CSS
  - Next.js 초기화 시 기본 제공되는 Tailwind를 활용하여 Notion 스타일의 레이아웃과 반응형 디자인을 빠르게 구현합니다.

## 🚀 구현 상세 계획

### 1. 프로젝트 초기화
- `kor-vibe-coding` 디렉토리에 Next.js 보일러플레이트를 설치합니다. (`npx create-next-app@latest .`)

### 2. 로컬 파일 시스템 연동 (API 라우트)
작성된 글을 실제 마크다운 파일로 로컬 디스크에 읽고 씁니다. (`/documents` 폴더 사용)
- `GET /api/files`: `/documents` 폴더 안의 `.md` 파일 목록을 읽어옵니다.
- `GET /api/files/[filename]`: 선택한 마크다운 파일을 읽어서 에디터에 로드합니다.
- `POST /api/files`: 에디터에서 작성한 내용을 마크다운 문자열로 변환하여 로컬 디스크의 `.md` 파일로 저장합니다.

### 3. 화면 레이아웃 (Notion UI)
- **좌측 사이드바**: 파일 목록 표시, '새 문서 생성' 버튼. (Notion의 좌측 메뉴와 동일한 느낌)
- **메인 에디터 영역**: BlockNote 라이브러리를 활용한 단일 페이지 에디터. 중앙 정렬 및 여백을 넉넉히 두어 세련된 읽기/쓰기 경험 제공.

### 4. 개발 워크플로우
1. Next.js 및 에디터 라이브러리 설치
2. 서버 측 API 구현 (로컬 파일 읽기/쓰기)
3. 좌측 사이드바 및 레이아웃 컴포넌트 개발
4. BlockNote 에디터 통합 및 파일 데이터 바인딩
5. 자동 저장(Auto-save) 또는 수동 저장 로직 추가 (Ctrl+S 지원)
