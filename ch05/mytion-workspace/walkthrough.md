# 구현 결과 요약 (Notion 스타일 WYSIWYG 에디터)

사용자님의 추가 요청에 따라 프로젝트 생성 위치를 변경하여 `ch05/mytion` 폴더에 Next.js 프로젝트를 무사히 구축했습니다. 또한, **화면 분할 없이 글을 작성하는 즉시 렌더링되는 Notion 스타일의 에디터**를 완벽히 연동했습니다.

## 🚀 개발 완료된 주요 기능

### 1. 📝 작성 즉시 렌더링 (BlockNote 에디터 적용)
- `@blocknote/core`, `@blocknote/react`를 활용하여 Notion과 똑같이 작동하는 단일 화면 블록 에디터를 만들었습니다.
- `#` 을 치고 스페이스바를 누르면 바로 제목으로 변하고, 슬래시(`/`)를 눌러 메뉴를 띄울 수 있습니다.
- 스크롤 동기화 문제를 고민할 필요 없이, 작성 중인 텍스트 커서 위치에 화면이 자연스럽게 고정됩니다.

### 2. 🗂️ 로컬 파일 디스크 저장 (API 라우트 연동)
- Node.js의 `fs` 모듈을 활용하여 `/documents` 폴더(루트 경로)에 실제 `.md` 파일로 읽고 쓰는 API를 구현했습니다. (`src/app/api/files`)
- 브라우저를 닫거나 새로고침해도 로컬 디스크의 데이터를 불러오기 때문에 글이 영구 보존됩니다.

### 3. 💾 자동 저장(Auto-save) 로직
- 글을 쓴 후 1초간 입력이 없으면 백그라운드에서 조용히 API를 호출해 문서가 자동 저장됩니다. 우측 상단에서 "저장 중..." 및 "저장됨" 상태를 확인할 수 있습니다.

### 4. 🧭 깔끔한 사이드바 UI
- `lucide-react`의 직관적인 아이콘을 사용하여 'Private Workspace' 좌측 메뉴를 디자인했습니다. 문서 추가 버튼과 파일 목록 탐색 기능이 탑재되어 있습니다.
- 현재 선택한 문서는 회색 배경으로 하이라이트됩니다.

## 🧐 사용 방법

현재 백그라운드에서 Next.js 개발 서버가 실행 중입니다!
웹 브라우저를 열고 아래 주소로 접속해보세요:
> **http://localhost:3000**

1. 좌측 상단의 **`+` 버튼**을 눌러 새 문서 이름을 입력합니다.
2. 새 문서가 열리면 에디터 안쪽을 클릭하고 자유롭게 글을 써보세요.
3. 글을 쓰면 자동으로 저장되며, 실제 `documents` 폴더 내에 마크다운 파일로 생성되는 것을 확인하실 수 있습니다.

> [!TIP]
> **마크다운 익스포트 오차 안내**
> BlockNote는 자체적인 '블록(Block)' 구조를 가지고 있어서 완벽한 순수 마크다운보다는 약간의 고유 구조를 가지지만, 저장할 때는 최대한 일반 마크다운 문자열로 변환하여(lossy 변환) `.md` 확장자로 저장합니다!
